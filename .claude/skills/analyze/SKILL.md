# /analyze - コード品質分析

プロジェクトのコード品質を分析し、改善点を提案します。

## 実行内容

1. ESLintによる静的解析
2. TypeScriptの型チェック
3. 依存関係の脆弱性チェック
4. バンドルサイズ分析
5. 改善提案の生成

## 使用方法

```
/analyze              # 全体分析
/analyze --security   # セキュリティ重点
/analyze --perf       # パフォーマンス重点
/analyze [path]       # 特定パスのみ
```

## 実行コマンド

```bash
# ESLint
npm run lint

# TypeScript
npm run type-check

# 脆弱性チェック
npm audit

# バンドル分析
npm run analyze
```

## 分析項目

### コード品質
- [ ] ESLintエラー・警告
- [ ] TypeScriptエラー
- [ ] 未使用のインポート・変数
- [ ] 複雑度の高い関数

### セキュリティ
- [ ] 依存パッケージの脆弱性
- [ ] 環境変数のハードコード
- [ ] SQLインジェクションリスク
- [ ] XSSリスク

### パフォーマンス
- [ ] バンドルサイズ
- [ ] 不要な依存関係
- [ ] 画像最適化
- [ ] SSR/SSG の適切な使用

## 出力形式

分析結果は以下の形式で出力されます:

```
## 分析結果サマリー

- ESLintエラー: X件
- ESLint警告: X件
- TypeScriptエラー: X件
- 脆弱性: X件（高: X, 中: X, 低: X）

## 改善提案

1. [高優先度] ...
2. [中優先度] ...
3. [低優先度] ...
```
