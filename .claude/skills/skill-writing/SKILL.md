# /skill-writing - スキル作成ガイド

新しいClaude Codeスキルの作成方法を説明します。

## スキルとは

スキルは、Claude Codeで使えるカスタムコマンドです。`/コマンド名` で呼び出せます。

## ディレクトリ構造

```
.claude/skills/
└── [skill-name]/
    └── SKILL.md      # スキル定義ファイル（必須）
```

## SKILL.mdの書き方

```markdown
# /[skill-name] - 短い説明

詳細な説明をここに書きます。

## 実行内容

このスキルが何をするかを箇条書きで説明します。

1. ステップ1
2. ステップ2
3. ステップ3

## 使用方法

\`\`\`
/skill-name              # 基本的な使い方
/skill-name --option     # オプション付き
/skill-name [引数]       # 引数付き
\`\`\`

## 実行コマンド

\`\`\`bash
# 実際に実行されるコマンド
npm run something
\`\`\`

## 前提条件

- 必要なツールやセットアップ
- 環境変数の設定など

## 注意事項

- 使用上の注意点
- 制限事項など
```

## スキル作成のベストプラクティス

### 1. 命名規則
- 短く、わかりやすい名前
- 動詞または動詞+名詞（例: deploy, test, create-component）
- ケバブケース（小文字、ハイフン区切り）

### 2. 説明文
- 最初の行に `/スキル名 - 簡潔な説明` を書く
- 何ができるかを明確に
- 使用例を含める

### 3. コマンド
- 実行されるコマンドを明記
- オプションや引数の説明を含める
- エラー時の対処法も記載

### 4. 前提条件
- 必要なツール・依存関係
- 環境変数
- 権限

## スキルの例

### シンプルなスキル
```markdown
# /format - コードフォーマット

Prettierでコードをフォーマットします。

## 実行コマンド
\`\`\`bash
npm run format
\`\`\`
```

### オプション付きスキル
```markdown
# /build - プロジェクトビルド

## 使用方法
\`\`\`
/build              # 開発ビルド
/build --prod       # 本番ビルド
/build --analyze    # バンドル分析付き
\`\`\`

## 実行コマンド
\`\`\`bash
# 開発
npm run build

# 本番
npm run build -- --mode production

# 分析
npm run build -- --analyze
\`\`\`
```

## 新しいスキルを追加する手順

1. `.claude/skills/[skill-name]/` ディレクトリを作成
2. `SKILL.md` ファイルを作成
3. 上記のテンプレートに従って内容を記述
4. Claude Codeで `/skill-name` を実行してテスト
